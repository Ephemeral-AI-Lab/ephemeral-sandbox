export class BackgroundSessionSupervisor

Generic background-session lifecycle, keyed by the native ids the model already holds. Background sessions are loop lifecycle: the loop's exit path calls `dispose()` on every finish, and every terminal transition publishes one `session_settled` notification whose drain marks the session delivered - delivery bookkeeping never leaves this class.
