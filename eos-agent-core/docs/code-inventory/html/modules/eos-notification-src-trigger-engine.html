export class NotificationTriggerEngine implements LoopObserver

The runtime's `LoopObserver` (Phase 04.9): runs operator trigger rules against loop lifecycle facts and publishes their answers as `{type: "reminder"}` notifications into the run's inbox. Notification-only: scripts inform, the model acts. Never throws or rejects — every script failure is logged and the firing dropped, matching the inbox's `onDrained` precedent. Timers live here so the engine stays clock-free; a generation counter makes idle-timer fires atomic per park epoch.
