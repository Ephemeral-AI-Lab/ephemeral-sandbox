# eos-command-session

## Overview

`eos-command-session` owns explicit live E2E coverage for the command-session daemon contract across protocol smoke, core lifecycle, ephemeral workspace process-group behavior, and isolated workspace command cleanup. It exercises `api.v1.exec_command`, `api.v1.write_stdin`, `api.v1.command.collect_completed`, `api.v1.command.cancel`, `api.v1.command_session_count`, `api.v1.cancel`, and `api.layer_metrics`. Module config lives at `crates/eos-e2e-test/tests/eos-command-session/config/default.test.yml`.

## Checklist

- [ ] eos-command-session-lifecycle: `exec_command`, `write_stdin`, collect-completed, cancel, timeout, session count, and output cap behavior are coherent for running and completed sessions.
- [ ] eos-command-session-cursors: Command output cursors advance without replaying previously consumed output across repeated polls or stdin writes.
- [ ] eos-command-session-terminate-kills-group: Command termination cancels the owned process group, drains command sessions, releases leases, and leaves no descendant marker processes.
- [ ] eos-command-session-detached-child-contract: `nohup` and `setsid nohup` descendants have an explicit tested contract instead of an accidental leak policy.
- [ ] eos-command-session-ephemeral-process-group: Ephemeral workspace command sessions stay running until all same-pgid subprocesses exit and collect exactly one terminal result.
- [ ] eos-command-session-isolated-cleanup: Isolated workspace command cancellation and exit discard mode make same-port re-entry reusable.
- [ ] eos-command-session-command-matrix: At least ten shell command families run through `exec_command` with multiple variants, foreground completion, changed-path assertions, and bounded timing/resource telemetry.
- [ ] eos-command-session-prompt-read-cancel: Prompt-style stdin input, empty cursor polling, collect-completed reads, and cancellation behave coherently without replaying consumed output.
- [ ] eos-command-session-parallel-load: Command sessions execute under the configured `1/3/6/12` parallel ladder without leaked sessions, leaked leases, unbounded upperdir/run-dir growth, or lost publishes.
- [ ] eos-command-session-parallel-prompt-load: Prompt-style command sessions run at the configured `1/3/6/12` ladder with isolated stdin payloads, no cursor replay, clean cancellation, and zero leaked sessions or leases.
- [ ] eos-command-session-negative-invalid: Nonzero exits, missing commands, unknown session ids, bogus stdin/cancel requests, and collect-completed misses return structured payloads without leaking sessions or leases.
- [ ] eos-command-session-output-backpressure: Large PTY output bursts honor `max_output_tokens`, preserve UTF-8 boundaries across cursor polls, and remain cancellable without leaked sessions or leases.
- [ ] eos-command-session-isolated-prompt-parity: Isolated command sessions accept prompt input, support empty polling and collect-completed reads, cancel cleanly, and discard private command writes on isolated exit.
