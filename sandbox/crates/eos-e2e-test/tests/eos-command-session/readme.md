# eos-command-session

## Overview

`eos-command-session` owns explicit live E2E coverage for the command-session daemon contract across protocol smoke, core lifecycle, transcript-log recycling, ephemeral workspace process-group behavior, externally killed process groups, invisible background subprocesses, and isolated workspace command cleanup. It exercises `api.v1.exec_command`, `api.v1.write_stdin`, `api.v1.command.collect_completed`, `api.v1.command.cancel`, `api.v1.command_session_count`, `api.v1.cancel`, and `api.layer_metrics`. Module config lives at `crates/eos-e2e-test/tests/eos-command-session/config/default.test.yml`.

## Checklist

- [ ] eos-command-session-lifecycle: `exec_command`, `write_stdin`, collect-completed, cancel, timeout, session count, and timestamped transcript behavior are coherent for running and completed sessions.
- [ ] eos-command-session-transcript-progress: Timestamped transcript progress remains observable through read_progress and operation-scoped stdin responses.
- [ ] eos-command-session-transcript-recycling: PTY transcript logs are transient capture files and are removed after finite pre-yield completion, collect-completed finalization, cancellation, external process death, parallel prompt cleanup, and isolated command cancellation.
- [ ] eos-command-session-cancel-kills-group: Command cancellation tears down the owned process group, drains command sessions, releases leases, and leaves no descendant marker processes.
- [ ] eos-command-session-teardown-control-cancel: Ctrl-C (`\u0003`) and Ctrl-D (`\u0004`) stdin payloads both route to command-session cancel, drain the session, and recycle the transcript log.
- [ ] eos-command-session-detached-child-contract: `nohup` and `setsid nohup` descendants have an explicit tested contract instead of an accidental leak policy.
- [ ] eos-command-session-ephemeral-process-group: Ephemeral workspace command sessions stay running until all same-pgid subprocesses exit and collect exactly one terminal result.
- [ ] eos-command-session-isolated-cleanup: Isolated workspace command cancellation and exit discard mode make same-port re-entry reusable.
- [ ] eos-command-session-command-matrix: At least ten shell command families run through `exec_command` with multiple variants, foreground completion, changed-path assertions, and bounded timing/resource telemetry.
- [ ] eos-command-session-prompt-read-cancel: Prompt-style stdin input, read_progress tail reads, collect-completed reads, and cancellation behave coherently without replaying consumed output.
- [ ] eos-command-session-long-running-stderr-stdin: Long-lived command sessions that emit stderr, consume stdin, and then keep sleeping remain running and uncollectable until terminal action.
- [ ] eos-command-session-external-process-death: Externally signaled children and process groups finalize or become observable through `write_stdin` without leaking sessions, leases, or marker processes.
- [ ] eos-command-session-invisible-background-process: Silent redirected same-pgid subprocesses keep command sessions running even after the foreground command exits, preventing invisible background work from being missed.
- [ ] eos-command-session-parallel-load: Command sessions execute under the configured `1/3/6/12` parallel ladder without leaked sessions, leaked leases, unbounded upperdir/run-dir growth, or lost publishes.
- [ ] eos-command-session-parallel-prompt-load: Prompt-style command sessions run at the configured `1/3/6/12` ladder with isolated stdin payloads, transcript-scoped replies, clean cancellation, and zero leaked sessions or leases.
- [ ] eos-command-session-negative-invalid: Nonzero exits, missing commands, unknown session ids, bogus stdin/cancel requests, and collect-completed misses return structured payloads without leaking sessions or leases.
- [ ] eos-command-session-output-backpressure: Large PTY output bursts stay timestamped in the transcript-backed stream and remain cancellable without leaked sessions or leases.
- [ ] eos-command-session-stdin-backpressure: Bounded stdin writes to non-reading consumers return promptly, leave the session cancellable, drain resources, and recycle transcript logs.
- [ ] eos-command-session-isolated-prompt-parity: Isolated command sessions accept prompt input, support read_progress snapshots and collect-completed reads, cancel cleanly, and discard private command writes on isolated exit.
