10. Workflow Cookbook §

  • This section restates the main runtime paths as end-to-end traces so a reader can follow one request from API/tool entrypoint to storage effect.
  • Each trace names the component that owns the next state transition instead of describing the architecture as abstract boxes.
  • The traces are intentionally redundant with earlier sections: they are meant to be useful while debugging a concrete workflow.

10.1 Direct LayerStack Read §

  1. The daemon router receives a single-path read_file request and confirms no isolated handle is active for the caller.
  2. The router converts the sandbox path to a workspace-relative path through the current WorkspaceBinding.
  3. LayerStack reads the active manifest and asks MergedView for the file content.
  4. MergedView searches newest-first through layer indexes, applying whiteout/opaque semantics during lookup.
  5. The router returns content with direct fast-path timings and zero overlay resource metrics.

Relevant code: router fast path backend/src/sandbox/daemon/tool_call_router.py:70-88, path binding backend/src/sandbox/daemon/tool_call_router.py:184-197, direct resource metrics backend/src/sandbox/daemon/tool_call_router.py:205-234, and merged-view reads backend/src/sandbox/layer_stack/view.py:60-114.

10.2 Ephemeral Shell or Search Operation §

  1. The router selects the ephemeral pipeline because the caller has no open isolated handle.
  2. The pipeline optionally performs shell pre-mount maintenance when manifest depth is high.
  3. Overlay lifecycle leases the active LayerStack snapshot and allocates one run dir with upper/work.
  4. The namespace runner launches a fresh user/mount namespace and sends the operation payload to the namespace entrypoint.
  5. The entrypoint mounts overlayfs onto the workspace root and runs the primitive command inside that view.
  6. For read-only intent, the parent releases the overlay handle without publishing.
  7. For write intent, the parent captures upperdir changes, converts them to OCC changes, publishes if accepted, emits workspace events, and destroys the run dir.

Relevant code: pipeline run loop backend/src/sandbox/ephemeral_workspace/pipeline.py:131-179, shell pre-mount maintenance backend/src/sandbox/ephemeral_workspace/pipeline.py:219-250, namespace entrypoint backend/src/sandbox/overlay/namespace_entrypoint.py:85-175, and publish helper backend/src/sandbox/ephemeral_workspace/workspace_publish.py:157-217.

10.3 OCC Publish from API Write/Edit §

  1. The router converts write_file or edit_file arguments into typed OCC changes.
  2. OCC prepare normalizes the path, checks gitignore state from the snapshot, and attaches base hashes for gated paths.
  3. The prepared changeset enters the commit queue, which keeps publish work serialized and only batches safe disjoint requests.
  4. The transaction reads the current active manifest, revalidates the path group, writes accepted content into OCC staging, and builds layer deltas.
  5. The publisher appends one immutable layer and writes the new manifest if the expected manifest still matches.
  6. The daemon projects OCC statuses into public tool result conflicts and published path metadata.

Relevant code: direct write/edit router backend/src/sandbox/daemon/tool_call_router.py:91-181, preparer backend/src/sandbox/occ/changeset_preparation.py:41-174, commit queue backend/src/sandbox/occ/commit_queue.py:55-239, transaction backend/src/sandbox/occ/commit_transaction.py:65-141, and result projection backend/src/sandbox/daemon/changeset_projection.py:16-60.

10.4 Isolated Workspace Tool Lifecycle §

  1. enter_isolated_workspace refuses to start while sandbox-bound background work is in flight for the agent.
  2. The daemon isolated pipeline prepares one LayerStack snapshot and stores the lease on the handle.
  3. The runtime creates a persistent namespace holder, veth network, DNS config, cgroup, and overlay mount.
  4. Every subsequent tool call routes to IsolatedPipeline, executes inside the existing namespace, and writes to the same private upperdir.
  5. Tool calls capture changed paths for audit only; they do not publish through OCC.
  6. exit_isolated_workspace cancels sandbox-bound background work, tears down the namespace resources, releases the lease, measures discarded bytes, and deletes scratch.

Relevant code: tools registry backend/src/tools/sandbox/_lib/registry.py:17-31, enter tool backend/src/tools/isolated_workspace/enter_isolated_workspace/definition.py:24-47, exit tool backend/src/tools/isolated_workspace/exit_isolated_workspace/definition.py:24-46, handle lifecycle backend/src/sandbox/isolated_workspace/_control_plane/handle_lifecycle.py:21-258, and isolated run path backend/src/sandbox/isolated_workspace/pipeline.py:121-158.

10.5 Plugin Write Operation §

  1. Host dispatch resolves plugin manifest, installs/refreshes the bundle, and ensures daemon runtime for the plugin digest.
  2. The daemon plugin registry dispatches a WRITE_ALLOWED op through the generic workspace overlay wrapper unless the plugin opted out.
  3. The wrapper acquires an operation overlay from the ephemeral pipeline and runs the plugin child in a fresh namespace.
  4. The child sees the workspace root as the overlay projection and writes only into that operation's upperdir.
  5. The daemon parent captures the upperdir, publishes through publish_cycle(), attaches publish metadata, and destroys the overlay.

Relevant code: host dispatch backend/src/sandbox/ephemeral_workspace/plugin/host_dispatch.py:70-186, plugin registry backend/src/sandbox/ephemeral_workspace/plugin/op_registry.py:85-220, operation overlay dispatch backend/src/sandbox/ephemeral_workspace/plugin/overlay_dispatch.py:29-173, and workspace publish cycle backend/src/sandbox/ephemeral_workspace/workspace_publish.py:84-99.

10.6 Pyright Latest-Projection Refresh §

  1. A read-only LSP tool asks the session manager for the current session.
  2. The manager compares the session's manifest key with the active key from projection or overlay.
  3. If different, it acquires or remounts a private overlay handle so Pyright sees the latest LayerStack projection.
  4. The ephemeral pipeline's workspace event bus also sends publish/full-resync events to the session manager.
  5. Pyright receives watched-file notifications, closes deleted open documents, and syncs changed open documents.
  6. LSP write operations, such as applying workspace edits, use a separate operation overlay and publish through OCC.
