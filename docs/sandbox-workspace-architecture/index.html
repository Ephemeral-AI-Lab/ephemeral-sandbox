Wiki Contract This directory is the maintained wiki layer over the sandbox source tree. The code checkout is the raw source of truth; these pages synthesize structure, workflows, invariants, and backlinks so a future reader does not have to rediscover the architecture from scratch.

Schema

Maintenance loop
When code changes, update the smallest affected node, then update nearby backlinks and the search catalog. Do not bury contradictions: link the newer evidence and explain the mismatch.

Source Graph §

The current sandbox workspace runtime is best read as a graph of ownership boundaries rather than a stack of folders. The daemon receives tool/API envelopes, routes work into direct LayerStack/OCC paths, per-operation overlay paths, or isolated private scratch, and every durable mutation converges on OCC before LayerStack publishes a new manifest.

registerssandbox.daemon.rpc.dispatcher registers built-in workspace ops, isolated lifecycle ops, daemon control ops, and plugin runtime ops through OP_TABLE backend/src/sandbox/daemon/rpc/dispatcher.py:344-373.
callsdispatch_workspace_tool_call() chooses isolated pipeline first, then direct file fast paths, then the ephemeral overlay pipeline backend/src/sandbox/daemon/workspace_tool_dispatch.py:53-75.
owns-stateLayerStack owns active manifests, immutable layer directories, leases, merged reads, publish transactions, squash, and flush backend/src/sandbox/layer_stack/stack.py:69-343.
owns-stateIsolatedPipeline owns per-agent private handle maps, namespace holders, veth resources, TTL, persisted handle state, and JSONL audit events backend/src/sandbox/isolated_workspace/pipeline.py:44-188.
tested-byLive and mock suites cover workspace base, LayerStack/OCC/overlay, background tools, plugins, and isolated workspace behavior under backend/tests/live_e2e_test/sandbox and backend/src/task_center_runner/tests/mock/sandbox.
