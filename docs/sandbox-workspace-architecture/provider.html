7. Provider §

  • The sandbox runtime can run on different backends without changing workspace logic.
  • The provider boundary defines provider-neutral CRUD, exec, preview/log, and context-preparation primitives.
  • Host lifecycle bootstraps runtime and workspace base through the selected adapter.
  • Providers own containers; sandbox workspace logic stays provider-agnostic.

ProviderAdapter is the single protocol for Docker and Daytona. It includes health/discovery, container CRUD, preview/build logs, async exec, and context_preparer() backend/src/sandbox/provider/protocol.py:21-79. The registry supports one process-wide default plus explicit per-sandbox bindings; fallback to default is intentionally uncached backend/src/sandbox/provider/registry.py:1-95. Startup chooses EOS_SANDBOX_PROVIDER or central config and bootstraps Docker or Daytona once backend/src/sandbox/provider/bootstrap.py:33-77.
