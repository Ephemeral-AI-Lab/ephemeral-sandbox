9. Space Model §

  • The design avoids copying a large repository for every tool call or background job.
  • Immutable lower layers are shared; private upper/work scratch is allocated per operation or isolated session.
  • LayerStack leases pin shared layers, overlay upperdirs hold changed bytes, and OCC publishes compact deltas.
  • Concurrent work scales with changed data, not with repository size.

Lower layers are referenced by path and lease; each concurrent operation adds only its own upper and work directories.

For normal ephemeral operations and isolated enter, additional lowerdir storage is O(1) repository bytes per operation: code returns existing layer_paths and a lease, not a rendered copy of the tree backend/src/sandbox/layer_stack/stack.py:105-129. The mount still carries O(manifest depth) path references and the shell path can pre-squash deep manifests before mounting backend/src/sandbox/ephemeral_workspace/pipeline.py:221-230, but it does not allocate O(repository bytes) lowerdir space.

For n concurrent operations, writable scratch is O(n * per-operation changed bytes) plus small metadata. Each operation/session has its own upper and work directories backend/src/sandbox/overlay/writable_dirs.py:46-52. Capturing changes walks only the upperdir and emits writes/deletes/symlinks/opaque markers backend/src/sandbox/overlay/capture.py:19-93. OCC then publishes accepted changes as one new immutable layer rather than copying the whole workspace backend/src/sandbox/occ/commit_transaction.py:113-141.
