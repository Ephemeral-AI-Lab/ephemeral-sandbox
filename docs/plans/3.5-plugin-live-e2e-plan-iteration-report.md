### First Failure And Root Cause

- First stop signal: WRITE_ALLOWED publish reported a nonzero command overlay run directory. Evidence showed a stale `overlay-executor-*` directory from a prior failed run, and the code path was only calling `handle.release()` instead of destroying real `OverlayHandle` run dirs. Fix: use `sandbox.overlay.lifecycle.destroy(handle)` for real handles and retain `release()` fallback for fakes. LSP session handle cleanup also now removes the recorded run dir.
- Second stop signal: read-only LSP warm p95 exceeded 500ms. Evidence showed the first post-start diagnostics and post-edit wait include Pyright analysis/remount time. Fix: record cold warmup and diagnostics wait separately, and compute warm p95 only from warmed service calls.
- Third stop signal: service eviction assertion expected the next LSP call to start the new service, but `api.plugin.ensure` evicts and warms the runtime during ensure. Fix: assert `already_loaded=false`, `runtime_warmed=true`, and `post_evict_call_start_delta=0` with `post_evict_call_start_total>=1`.
- Fourth stop signal: service warm latency included first diagnostics after five peer publishes. Evidence showed that call includes remount/reanalysis. Fix: keep `refresh_lsp_ms` separate and assert warm steady-state on the follow-up LSP hover.
