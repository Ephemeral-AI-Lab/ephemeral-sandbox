### Optimize workspace upload for layerstack / Layerstack Optimization

**Artifact:** [[docs/layerstack/create_sandbox|Create Sandbox — Layerstack Workflow & Optimizations]]
(`docs/obsidian/ephemeral-os/docs/layerstack/create_sandbox.md`),
`crates/sandbox-runtime/layerstack/src/workspace_base/layer.rs`,
`crates/sandbox-provider-docker/src/runtime.rs`

**Session:** Optimized the `create_sandbox` base-layer build (the workspace
upload into layerstack). Found the build is bound by per-file VirtioFS I/O
latency on a 4 vCPU / 4 GB Docker VM, not CPU. Applied, each verified to keep
`root_hash` byte-identical (`c66f29da14bb…`):

- **Parallel copy + hash** with a per-worker reused buffer (killed ~11 GB of
  per-file buffer zeroing) and dropped redundant per-file `create_dir_all` /
  `remove_path`.
- **lstat fold** — entry type from `readdir` `d_type`, permissions from the
  worker's `fstat` (~11k fewer round-trips; removed the serial-walk stall).
- **Off tmpfs** — moved `/eos/layer-stack` to disk; frees scarce VM RAM, fixes
  the multi-sandbox OOM, and lifts the 2 GB tmpfs import ceiling.
- **I/O-concurrency pool P=32** (sized for in-flight VirtioFS reads, not cores)
  — the biggest single win.
- **Exact-size read** (drop the trailing EOF probe) and **parallel directory
  walk** (overlap the ~1,697 serial `readdir`s).

Result: base-build ~5–6 s → **~1.6 s**, end-to-end `create_sandbox`
~6.7 s → **~2.1 s** (~3×), all format-compatible. BLAKE3 was investigated and
rejected (I/O-bound build + HW-accelerated SHA-256 + persisted-hash format
break → ~0–5% gain, inside noise).
