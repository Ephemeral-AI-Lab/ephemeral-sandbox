### daemon http

**Artifact:** spec `docs/daemon-http/README.md`; impl
`crates/sandbox-daemon/src/http/` (`server`/`router`/`response`/`health` +
`forward/{route,proxy}`) and `crates/sandbox-daemon/src/rpc/` (renamed from
`src/server/`); cross-crate `crates/sandbox-config/src/configs/manager.rs`
(`daemon_http_port`), `crates/sandbox-manager/src/model.rs`
(`SandboxHttpEndpoint`) + `daemon_install.rs` (`StartedDaemon`),
`crates/sandbox-provider-docker/src/engine.rs` (multi-port publish),
`crates/sandbox-runtime/operation/src/workspace_session/service/core.rs`
(`isolated_ip`); e2e
`cli-operation-e2e-live-test/runtime/daemon_http/test_daemon_http.py`

**Session:** Implemented daemon HTTP v0 to the spec тАФ expose in-sandbox HTTP
services through one Docker-published loopback port, without publishing arbitrary
service ports.

- **Config + record:** added `manager.docker.daemon_http_port` (default `7001`,
  validated `1..=65535`). Docker publishes both `daemon_port` and
  `daemon_http_port` to random `127.0.0.1` host ports through one shared
  multi-port publish helper (no duplicated Docker port logic); the sandbox record
  exposes `daemon_http {host, port}` beside `daemon`.
- **Listener:** a second HTTP listener in `sandbox-daemon`, separate from the
  JSON-line RPC transport тАФ no sniffing or multiplexing. Renamed `src/server/` тЖТ
  `src/rpc/` so the two transports sit side by side.
- **Routes:** `GET /health` (fixed JSON, never reads runtime state),
  `ANY /forward/shared/<port>/...`, `ANY /forward/isolated=<workspace_id>/<port>/...`.
- **Proxy (hyper HTTP/1.1):** preserves method/headers/body/query, strips the
  route prefix, streams both bodies, tunnels WebSocket/upgrade requests, and
  appends `X-Forwarded-Host` / `-Proto: http` / `-Prefix`. Shared resolves to
  `127.0.0.1:<port>`; isolated resolves to the workspace veth IP via runtime
  session state (`WorkspaceSessionService::isolated_ip`).
- **Errors mapped once** at the HTTP boundary via a single `ForwardError`: `400`
  invalid route / invalid port, `403` workspace has no reachable IP, `404`
  unknown isolated workspace / non-`/forward`, `502` connect failed, `504`
  timeout.
- **Observability:** one `daemon_http.forward` span per request through the
  existing `sandbox-observability` pipeline (route_kind, workspace_id, target,
  method, path_prefix, status_code, duration_ms, bytes_in/out, error_kind).

**Verification:** `cargo build` / `clippy --all-targets` / `fmt` clean (no new
violations, no `unwrap_used`); `sandbox-daemon` **50** tests pass. Live Docker
E2E **4/4 pass** тАФ `test_daemon_http_health`,
`test_forward_shared_arbitrary_port`, `test_forward_isolated_arbitrary_port`,
`test_forward_rejects_invalid_routes` тАФ each driving real sandboxes via
`sandbox-cli manager` + `runtime exec_command` (success tests bind port `0` and
use the assigned port), with every sandbox / workspace session / command cleaned

(http module 724 тЖТ 675 LOC). ~870 production LOC, no crate-boundary crossings.

### O(1) shared workspace base adoption spec

**Artifact:** [[docs/layerstack/o1_shared_workspace_base|O(1) Shared Workspace Base For N Sandboxes]]
(`docs/obsidian/ephemeral-os/docs/layerstack/o1_shared_workspace_base.md`)

**Session:** Drafted the adoption spec for reducing identical sandbox base repo
storage from `O(n)` to `O(1)`: build the workspace base once with the existing
parallel single-pass copy+hash algorithm, publish it as an immutable shared
base keyed by layerstack root hash, and bind that base read-only into each
sandbox while keeping per-sandbox metadata and future layers private.

**Proof plan:** The spec includes live e2e command examples for the current
`sandbox-cli` `O(n)` baseline, the future shared-base `--count 5` path,
`exec_command` latency/mount checks to catch
deferred-copy implementations, read-only safety checks, host workspace mutation
isolation, and the raw Docker storage smoke test already measured on
2026-06-30 (`863M` shared base, `180.2kB` total writable data across five
containers).
