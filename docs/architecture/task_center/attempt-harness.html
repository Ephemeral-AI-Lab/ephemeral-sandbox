The attempt harness is the Attempt layer of Workflow -> Iteration -> Attempt: planner submission, persisted generator DAG, evaluator submission, then closed Attempt. It owns one Attempt, not the whole Workflow or a global agent orchestration layer.

Attempt State Machine #

  • AttemptStage is PLAN to GENERATE to EVALUATE to CLOSED.
  • AttemptStatus is running until the close path marks the attempt PASSED or FAILED.
  • AttemptFailReason distinguishes planner failure, generator failure, evaluator failure, and startup failure.
  • AttemptOrchestratorRegistry is process-local lookup for active attempts, used by launch/exhaustion and delegated-workflow close-report delivery.

Planner And DAG Persistence #

A planner submission is accepted only for the current attempt planner task while the attempt is in PLAN. Full plans cannot set deferred_goal_for_next_iteration; deferring plans must set it. After validation, the orchestrator marks the planner task done, persists the plan contract, persists generator rows, records their ids on the attempt, moves the attempt to GENERATE, and advances ready work.

DAG validationordered_generator_tasks() rejects duplicate local ids, dependencies on unknown local ids, and dependency cycles before any runtime generator rows are used.
topological orderThe helper uses a stable Kahn sort. Runtime task ids are derived from attempt id plus planner local id, and dependency ids are normalized into persisted needs.
plan contractAttemptStoreProtocol.set_plan_contract() stores plan_spec, evaluation_criteria, and optional deferred-goal text. Later context recipes read that persisted state.
stage moveattempt_store.set_stage(..., AttemptStage.GENERATE) happens only after the planner task and generator rows have been persisted.

Generator Waves And Quiescence #

AttemptStageAdvancer is store-driven. On every generator submission or launch failure, it lists persisted generator tasks for the attempt, launches every pending task whose dependencies are done, and otherwise summarizes whether the DAG is still active, failed, blocked, or fully done.

settleFailed launches re-enter advancement so remaining runnable siblings, unreachable descendants, or terminal closure can settle from store state.
quiescePending descendants of failed or blocked upstreams are unreachable but persisted; once runnable siblings finish, the attempt closes with GENERATOR_FAILED.
evaluateOnly when every generator is DONE does stage advancement create and launch the evaluator task.
waiting goalWAITING_WORKFLOW is not quiescent and not done. A parent generator waiting on a child goal keeps the parent attempt from advancing until a close report is delivered.
blocked descendantsPending descendants of FAILED or BLOCKED tasks remain persisted as pending not-started work. They are counted as unreachable for quiescence, not launched.
blocker semanticsGeneratorSubmission(outcome="blocker") marks the task BLOCKED. The attempt waits for runnable siblings, then fails with GENERATOR_FAILED.

Evaluator And Closure #

The evaluator is spawned exactly once after all generator tasks are done. _start_evaluator_stage() builds the evaluator launch, emits task-ready and task-launched audit events, persists the evaluator task row as running, records evaluator_task_id, moves the attempt to EVALUATE, and launches the evaluator.

evaluator successAn evaluator task marked DONE closes the attempt as PASSED. The iteration coordinator then closes the iteration as succeeded and either emits terminal success or success deferred.
evaluator failureAn evaluator task marked FAILED closes the attempt as FAILED with EVALUATOR_FAILED. The iteration coordinator decides retry or failed iteration based on budget.
close ownerAttemptOrchestrator._close_attempt() is the only attempt-closing path used by stage advancement. It deregisters the orchestrator before notifying the iteration coordinator.
retry handoffThe orchestrator never creates retry attempts. Retry is handled after closure by IterationAttemptCoordinator.handle_attempt_closed().
Boundary The attempt harness closes only one Attempt. Iteration retry, deferred continuation, nested child Goals, and Workflow closure live one level up in the Workflow Iteration Attempt page.

Launcher Failures And Exhaustion #

EphemeralAttemptAgentLauncher schedules role-specific agent runs. Terminal submission tools are expected to move the task out of RUNNING during the run. If the runner returns, crashes, returns None, or exits without a terminal submission while the task remains running, the launcher reports role-specific exhaustion back into the same orchestrator API.

exhaustionIf the task remains RUNNING, _report_exhaustion() calls the planner, generator, or evaluator failure path on the active orchestrator.
unownedIf the orchestrator is missing, _fail_unowned_attempt() marks the task failed, closes the attempt directly, and notifies a registered iteration coordinator when present.
no launcher retryThe launcher invokes the runner once per launch and does not pass the deleted max_terminal_retries kwarg. Query-loop soft warnings and overshoot tolerance own in-run recovery.
startup failuresPlanner startup failure in AttemptOrchestrator.start() deregisters the orchestrator and marks the planner task failed; IterationAttemptCoordinator owns closing the inserted attempt as STARTUP_FAILED.
role-parametrized exhaustionCurrent source uses one _report_exhaustion() branch tree plus _ROLE_FAIL_REASONS. The older per-role reporter helper table is intentionally gone.
evaluator spawn failureEvaluator compose or launch failure closes the attempt with EVALUATOR_FAILED. Generator launch failure marks the generator failed and closes with GENERATOR_FAILED.

Audit Events #

TaskCenter attempt advancement emits a small set of audit events through TaskCenterAuditEmitter. The emitter is a write-only facade over the shared audit sink; it shapes node identity and payload fields so stage advancement does not repeat audit plumbing.

task_center.task.readyEmitted before a generator or evaluator launch with satisfied dependency ids and the task identity payload.
task_center.task.launchedEmitted after a task row moves into running launch state, carrying run id, attempt id, task id, role, agent name, needs, and context packet id.
task_center.task.failedEmitted when stage advancement marks a launched task failed because agent launch failed.
audit sinkAttemptDeps.audit_sink defaults to NoopAuditSink, so production wiring can observe events without making attempt logic depend on a concrete recorder.

Evidence And Refresh #

Primary Code

attempt/orchestrator.py owns submission entry points and attempt close. attempt/stage_advancer.py owns generator and evaluator advancement. attempt/generator_dag.py owns DAG validation and quiescence.

Launch Code

attempt/launch.py owns agent scheduling, row mapping, exhaustion reporting, and AgentLaunchFactory. attempt/deps.py carries runtime dependencies and delegated-workflow parent task behavior.
