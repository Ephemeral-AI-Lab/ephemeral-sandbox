Submission tools are terminal adapters under backend/src/tools/submission. They validate model output, resolve TaskCenter context, and delegate lifecycle mutation to TaskCenter attempt, goal, and run services.

make_submission_tools() returns the submission tool set registered by the framework factory: planner submissions, executor submissions, verifier submissions, evaluator submissions, helper feedback/result tools, and exploration result submission.

Planner, Executor, Verifier, Evaluator #

Planner tools resolve an attempt-bound submission context, build a validated planner submission from plan_spec, evaluation_criteria, flat tasks, task_specs, and optional deferred_goal_for_next_iteration, then call AttemptOrchestrator.apply_plan_submission(). The planner decides whether the attempt can close the current goal or delivers a bounded slice that defers the next iteration.

Executor success and blocker tools resolve ExecutorSubmissionContext, which wraps the generator task row, runtime dependencies, and attempt context. Success calls apply_generator_submission() with outcome="success", artifacts, and generator_role="executor". Blocker calls the same orchestrator path with outcome="blocker".

Verifier tools are also generator submissions, but their payload identifies generator_role="verifier" and carries checks or issues. Evaluator tools use EvaluatorSubmission and call AttemptOrchestrator.apply_evaluator_submission(), which lets the attempt stage advancer close, retry, or progress according to the stored plan contract and lifecycle rules.

Execution Handoff and Delegated Workflows #

  1. submit_execution_handoff(goal=...) validates a nonblank goal and resolves an executor submission context for the current attempt-bound generator task.
  2. ExecutorSubmissionContext.start_delegated_workflow() constructs WorkflowStarter(runtime=...) and calls WorkflowStarter.start(prompt=goal, origin=WorkflowOrigin.task(task_id=current_task_id)).
  3. WorkflowStarter validates that the parent generator task is running and has no open delegated child goal, creates the child goal, initial iteration, and initial attempt, then marks the parent task as WAITING_WORKFLOW.
  4. The handoff tool returns metadata containing submission_kind="workflow_start", parent task id, parent attempt id, child goal id, initial iteration id, and initial attempt id. Because it is terminal and successful, the framework stamps is_terminal=True.
  5. When the delegated workflow later closes, WorkflowClosureReportRouter delivers the report to the parent task. For task-origin goals, the parent must still be WAITING_WORKFLOW; the router calls the registered parent task object.
  6. The parent task delegates to AttemptOrchestrator.apply_workflow_closure_report(), which resumes the waiting generator task as DONE on success or FAILED on failure, records the closure report in the task summary payload, and advances the attempt stage.

This flow is the TaskCenter handoff boundary: agents do not communicate peer-to-peer. The child goal is persisted state, and the parent resumes through a closure report delivered back into the orchestrator.

Helper and Explorer Results #

submit_advisor_feedback and submit_exploration_result are terminal tools, but they do not resolve an attempt context or call TaskCenter mutators. They return bounded output text plus metadata that the caller can interpret: advisor verdict and role, or explorer findings and references.

This distinction is intentional. Helper agents terminate their own helper run and provide structured metadata to the parent workflow, while the parent TaskCenter attempt state remains owned by the planner/executor/verifier/evaluator submission paths and the orchestrator.

Workflow Connections #

frameworkEvery submit_* function is a decorated BaseTool with is_terminal_tool=True; successful execution is stamped by execute_tool_once().
engineTerminal exclusivity and TOOL_STOP are enforced in engine.tool_call.dispatch and engine.query.loop, not in individual submission tools.
TaskCenterAttempt state changes are centralized in AttemptOrchestrator. Handoff state changes are centralized in WorkflowStarter, WorkflowClosureReportRouter, and AttemptOrchestrator.apply_workflow_closure_report().
sandboxSubmission tools use Intent.READ_ONLY. They do not perform workspace file mutations directly; generator work that touched the sandbox should already have gone through sandbox tools before terminal submission.
promptingLaunch-time terminal routing and the shared terminal descriptor registry decide which terminal options appear in TaskCenter task guidance and skills.
  • Submission tools are adapters, not orchestrators.
  • Parent generator handoff uses persisted goals and closure reports, not direct agent-to-agent calls.
  • Helper terminal outputs return metadata to their caller without mutating TaskCenter attempt state.
