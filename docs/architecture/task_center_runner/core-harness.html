The core harness has one purpose: drive task_center.start_task_center_run with a per-test database, audit-bus fanout, sandbox lease, and lifecycle hooks. Three modes (mock, real-LLM, benchmark) reach the same coroutine through one frozen dataclass.

Scope #

  • core/engine.py::run_pipeline is the single entrypoint. It knows nothing about MockSquadRunner, Daytona, or sweevo symbols; tests/unit_test/test_task_center_runner/test_no_core_imports.py pins that.
  • core/runner.py::run_scenario is a thin shim that builds the mock-mode RunConfig, calls run_pipeline, and rebuilds the legacy mock-rich RunReport.
  • core/bootstrap.py seeds production runtime state for the real-LLM path only — markdown agent registry, runtime stores, sandbox provider. Mock runs never call it.
  • Everything else in core/ is a typed protocol or dataclass — RunConfig, RunContext, PipelineReport, LifecycleHooks, SandboxProvisioner, TaskCenterStoreBundle.

RunConfig And Mode Deltas #

RunConfig is a frozen dataclass with five mode-varying seams and a small bag of shared options. RunContext is the narrow handle (config, bundle, bus) passed to lifecycle hooks and sandbox provisioners — they cannot reach engine internals.

run_pipeline Lifecycle #

run_pipeline(config) coroutine
   |
   | 1. refuse_dual_disable_when_isolated_workspace_enabled()
   |       — fail-fast on EOS_DAEMON_AUDIT_PULL_ENABLED=false
   |         AND EOS_AUDIT_STREAM_FALLBACK=false
   |         AND EOS_ISOLATED_WORKSPACE_ENABLED=true
   |
   | 2. if config.bootstrap: config.bootstrap()
   |       — seeds provider + runtime stores + agent registry
   |
   | 3. bundle = config.stores or create_per_test_task_center_stores()
   |    bus = AuditEventBus(); bus.subscribe(config.lifecycle.on_event)
   |
   | 4. ctx = RunContext(config, bundle, bus)
   |    await config.lifecycle.before_run(ctx)
   |
   | 5. lease = await config.sandbox.provision(ctx)
   |    run_dir = (config.run_dir_factory or _default_run_dir)(audit_dir, ctx)
   |
   | 6. AuditRecorder(run_dir, ...).start()
   |    runner = config.runner_factory(ctx)
   |    runner.bind_audit_recorder?(recorder)
   |
   | 7. handle = start_task_center_run(
   |       config=extras["runtime_config"] or SimpleNamespace(cwd=repo_dir),
   |       prompt, sandbox_id, on_agent_event, task/goal/iteration/attempt/
   |       context_packet stores, runner, sandbox_provisioner)
   |    bus.publish(RUN_STARTED)
   |
   | 8. await handle.launcher.wait_for_idle()
   |       (optionally wrapped in asyncio.wait_for(timeout=max_duration_s))
   |    on TimeoutError: cancel pending launcher tasks; lifecycle.on_aborted
   |    bus.publish(RUN_COMPLETED)
   |
   | 9. finally:
   |       await config.sandbox.release(lease)
   |       recorder.aclose() or .dispose()  (Phase 2.6 puller drain)
   |       if owns_stores: bundle.close()
   |       lifecycle_unsub()
   |
   | 10. asyncio.create_task(_write_perf_report_safe(...))
   |     report = PipelineReport(...)
   |     await config.lifecycle.after_run(ctx, report)
   |     return report
stream-bridge_on_agent_event wraps stream_bridge(bus, task_center_run_id, sandbox_fallback_enabled) + per-task MessageRecorder.emit. Audit recording from agent runs flows through this callback.
perf-reportThe final perf-report write is spawned as a named asyncio task perf_report:<tcrid> and attached to PipelineReport.performance_report_task so callers can await it.
coding-plan detectiontry_get_active_model_kwargs()["class_path"].startswith("providers.clients.coding_plan.") sets coding_plan_mode_active on the recorder so audit segregation works for that provider family.

LifecycleHooks Protocol #

One protocol, four entry points, three implementations.

before_run(ctx)Awaited once after stores + bus exist but before sandbox provisioning. SweevoLifecycle and ScenarioLifecycle are both no-ops here.
on_event(event)Subscribed to the audit bus at startup. ScenarioLifecycle.on_event appends to captured_events, mirrors EventType into MutableMockState.seen_events, decodes MOCK_* payloads into typed records, and fires post-hooks. SweevoLifecycle ignores events.
after_run(ctx, report)Awaited once after the recorder is disposed. May mutate report.lifecycle_extras. SweevoLifecycle materializes the active overlay onto the repo and writes sweevo_result.json + aggregate.jsonl.
on_aborted(ctx, reason)Awaited inside the timeout branch before final cleanup. SweevoLifecycle stashes the reason for after-run reporting.
  • NoopLifecycle is the default for real-LLM freeform runs that do not need lifecycle observation.
  • ScenarioLifecycle bridges hooks into HookSet.fire on every event, populates the legacy RunReport view, and decodes the four MOCK_* event payloads.
  • SweevoLifecycle owns the grading contract; it never observes mock events because the benchmark uses the production runner, not the mock.

TaskCenterStoreBundle #

The bundle reuses the project's shared SQLAlchemy engine via db.engine.initialize_db() and carves a fresh isolated store per test so concurrent tests do not collide.

PostgreSQL_create_postgresql_bundle creates a fresh schema (<prefix>_<uuid12>), recreates all tables in it, and routes ORM calls via execution_options(schema_translate_map={None: schema}). close() drops the schema.
SQLite_create_sqlite_bundle creates a per-test .db file next to the configured SQLite database (or :memory: when no path), enables WAL + foreign keys + 30s busy timeout. close() disposes the engine and unlinks the per-test file (plus -wal, -shm, -journal companions).
  • Stores in the bundle: task_store, workflow_store, iteration_store, attempt_store, context_packet_store.
  • Each store is constructed and then store.initialize(session_factory) is called so the same instance can be reused safely under one bundle.
  • Other dialects raise — only PostgreSQL and SQLite are supported.

SandboxProvisioner Protocol #

SandboxLeaseFrozen handle: sandbox_id plus opaque metadata mapping. Returned by provision(ctx); consumed by release(lease).
AttachExistingBuilds a lease around a pre-provisioned sandbox. release() is a no-op so a fixture-managed container survives the run. Used by every mock scenario and by tests that pre-create a sandbox.
SweevoProvisionerRe-runs setup_sweevo_sandbox inside provision() so retry attempts get a clean checkout, then returns a lease with instance_id/repo_dir metadata. release() is a no-op — the SWE-EVO container persists across runs.
  • The engine treats the lease opaquely — only lease.sandbox_id is read.
  • The default TaskCenterSandboxProvisioner used by start_task_center_run is a permissive shim built by _default_sandbox_provisioner that accepts the caller-supplied id; benchmark runs swap this out via config.sandbox_provisioner_factory.

Audit Bus And Recorder #

Two layers: the in-memory AuditEventBus for hook fanout, and AuditRecorder for durable per-run artifacts.

AuditEventBusSynchronous fanout, single-threaded, no locking. publish(event) calls every handler; handler errors are captured into bus.errors rather than raised. subscribe(handler) returns an unsubscribe callable.
Event typesEventType enumerates run lifecycle, agent invocation, terminal outcomes, tool-call, sandbox-derived (write/edit/shell/batch-edit/conflict, LayerStack lease/layer/squash, OCC commit, isolated workspace events), hook synthetic, and four mock side-channel events.
NodeIdEvery event carries a NodeId identifying the task_center_run_id, optional workflow_id, iteration_id, attempt_id, task_id. Subscribers use it to route per-task work.
AuditRecorderWrites the run-dir's persistent artifacts. Bound to a single task_center_run_id via bind_task_center_run_id. Exposes metrics (live snapshot) and metrics.performance_snapshot() consumed by the perf report.
Stream bridgestream_bridge(bus, task_center_run_id, sandbox_fallback_enabled) is the callback that turns engine-stream agent events into bus publications plus per-task message-recorder writes.
  • Phase-timing contract: sandbox_isolated_workspace_* events carry total_ms and phases_ms; phase keys are emitted only when that codepath actually ran. Aggregators consume them opaquely.
  • _refuse_dual_disable_when_isolated_workspace_enabled hard-fails startup when both audit paths (daemon pull, stream-bridge fallback) are disabled with EOS_ISOLATED_WORKSPACE_ENABLED=true — silent regression prevention from V3 Phase 3.

PipelineReport And RunReport #

PipelineReport is the engine's narrow return shape; the legacy mock-rich RunReport is rebuilt by run_scenario from ScenarioLifecycle records.

PipelineReport corestatus ("completed" / "aborted"), task_center_run_id, request_id, sandbox_id, instance_id, run_dir, task_center_status, duration_s, task_count, tasks_completed, tasks_failed, metrics, aborted_by_timeout.
PipelineReport extraslifecycle_extras: dict — SweevoLifecycle.after_run writes sweevo_result here. performance_report_task is the async handle for the perf-report write.
RunReport additionsMock-only fields rebuilt from ScenarioLifecycle: events, seen_event_types, hook_results, mutable_state_flags, launches, tool_calls, prompt_inspections, sandbox_checks. Plus graph_summary built by _graph_summary(bundle, tcrid) while the stores are still open.
  • The shim closes its owned bundle only AFTER _graph_summary reads goals → iterations → attempts → tasks, so the dict-shaped graph stays consistent.
  • passed_prompt_inspections and passed_sandbox_checks are convenience properties on RunReport that AND-together every recorded check.
