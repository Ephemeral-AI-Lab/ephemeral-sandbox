This wiki maps how an EphemeralOS agent run is assembled, executed, monitored, and terminated. It connects the engine loop, model-provider streaming, tool framework, sandbox-provider boundary, TaskCenter context engine, notifications, and message artifacts.

Scope And Verdict #

  • The main agent loop is engine-owned. TaskCenter submits and resumes state through terminal tools and persisted attempt/goal state, not peer-to-peer agent messaging.
  • The model-provider package is backend/src/providers. It owns the LLM API request/streaming contract, active model dispatch, auth strategies, provider errors, and Anthropic/Codex client translations.
  • Prompt configuration has two layers: stable runtime/profile system prompt text, then TaskCenter launch rows built from context packets, task guidance, and optional skill text.
  • Tool execution crosses into sandbox through typed tool wrappers and sandbox.api. Normal file and shell operations use daemon workspace routing; provider raw exec is a transport and diagnostic primitive.
  • Termination is a stamped tool-result condition. A successful terminal tool gets is_terminal=true, the query loop exits with TOOL_STOP, and TaskCenter lifecycle code reads the structured submission.

Staleness Notes #

  • The older docs/task_center_harness_and_context_engine.html contains stale context-engine claims: it still mentions entry_executor, says executor terminal routing is no longer depth-filtered, and describes renderer compression as if high-priority blocks can be dropped.
  • Current source says built-in TaskCenter context recipes are planner, generator, and evaluator; executor terminal handoff is still filtered at nested workflow depth greater than one; renderer truncates low then medium priority blocks, never required or high.
  • The current agent lifecycle is single-pass. run_ephemeral_agent does not retry with a fresh budget; missing terminal output becomes exhaustion/failure handling at the TaskCenter launch boundary.
  • Historical live reports may mention api.shell_batch. Current daemon/API source exposes api.v1.shell plus harness-local batch helpers, not a public shell-batch daemon operation.
  • The model-provider package is plural: backend/src/providers. The singular backend/src/provider path does not exist in this checkout.
