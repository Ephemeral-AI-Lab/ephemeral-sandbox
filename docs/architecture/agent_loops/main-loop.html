Provider Turn Workflow

The query loop owns one provider-visible turn at a time. It prepares tool metadata, background supervision, terminal-tool names, notification state, provider-safe history, and prompt-report rows before the model-provider stream starts.

The durable transcript remains local to the agent. Provider requests receive a sanitized copy from build_provider_messages(), which compacts stale background snapshots and drops malformed tool-use/result pairs without mutating the run transcript.

Streaming and Deferral

StreamingToolExecutor can execute normal foreground tools as soon as tool-use deltas arrive. The executor itself does not know background or terminal policy; the query loop passes a should_defer predicate.

Terminal-capable runIf context.terminal_tools is non-empty, every streamed tool is deferred until the final assistant message is available. This lets the final batch validator reject a terminal tool batched with siblings before any sibling side effect runs.
Background-bound calldefer_background_dispatch() returns true for engine-background policy tools such as run_subagent. The final dispatcher later launches them through the supervisor path.
Foreground callNon-deferred tools start mid-stream, emit lifecycle/progress events, and are gathered by executor.get_remaining() before final dispatch decides which final-message tool ids remain unresolved.
Failure modeIf the provider stream ends without an AssistantMessageCompleteEvent, the executor cancels in-flight streamed tools and the loop raises a provider-facing runtime error instead of dispatching a partial assistant message.

Termination

The agent loop has one job: keep looping until a terminal tool is submitted. The only failure condition is reaching ceil(1.5 × tool_call_limit) tool calls without a terminal submission.

Two global invariants make this design minimal:

  • Every agent declares at least one terminal tool (AgentDefinition.terminals non-empty).
  • Every agent declares an integer tool-call budget (AgentDefinition.tool_call_limit: int).

Exit reasons:

  • TOOL_STOP — a terminal-tool dispatch returned is_terminal=True. The terminal tool's ToolResult is the run's deliverable.
  • TERMINAL_NOT_SUBMITTED — the run hit tool_calls_used >= ceil(1.5 × tool_call_limit) without a terminal submission. Background tasks are cancelled and a synthetic ToolExecutionCompletedEvent(is_error=True) is emitted on the stream.

Default notifications are split by purpose. The terminal_tool_call_count_reminder rules fire once at 75%, 100%, and 125% of tool_call_limit while terminal_result is None. The repeating terminal_call_reminder still fires every turn after the first assistant message while no terminal result exists. Both reminder families include the current used/limit budget, the hard ceiling, and the remaining turns until failure.

Transcript invariant: on the TERMINAL_NOT_SUBMITTED exit, if the failing assistant turn produced tool_use blocks, the paired tool_result blocks are appended to the transcript before the loop breaks — preserving the Anthropic Messages API contract.

Run Lifecycle Boundary

run_ephemeral_agent() is the single lifecycle entry point for top-level agent runs and subagent launches. It spawns the agent, drives agent.run(..., auto_close=False), forwards stream events, persists the agent run, closes the provider client, and returns an EphemeralRunResult.

There is no retry-with-fresh-budget mechanism at this layer. If the query loop exits without a terminal result because of TERMINAL_NOT_SUBMITTED or a crash, run_ephemeral_agent() returns no terminal result; only unhandled exceptions set status="failed" directly through run_error.

Soft recovery remains in-band. The factory attaches default budget-tier warnings plus the repeating terminal_call_reminder for every agent. The reminders are user-message blocks in the same transcript, not hidden outer retries.

            QueryExitReason                EphemeralRunResult                  Caller view
            ----------------                -------------------                 ------------
            TOOL_STOP               ---->  terminal_result = ToolResult        success
            TERMINAL_NOT_SUBMITTED  ---->  terminal_result = None              soft fail (in-band)
            unhandled exception     ---->  status = "failed", run_error set    hard fail

            * No fresh-budget retry. Budget-tier warnings and
              terminal_call_reminder nudge the agent in-band; the hard ceiling
