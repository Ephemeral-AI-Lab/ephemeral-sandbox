This support page records how the Agent Loops wiki was built and how to keep it current. It is documentation metadata only; runtime code is not owned here.

Refresh Triggers #

Main Loop

Refresh when query-loop exit reasons, terminal-tool dispatch, provider-history sanitation, prompt-report recording, or tool budget behavior changes.

Background Operations

Refresh when background metadata, aliasing, check/wait/cancel output, progress providers, sandbox heartbeat/cancel, daemon in-flight registry, or audit event names change.

Prompt And Context

Refresh when profile frontmatter fields, terminal filtering, context recipes, renderer tag/compression policy, task guidance, skills, or TaskCenter retry/deferred lifecycle changes.

Provider Runtime

Refresh when provider bootstrap, registry fallback, raw exec, daemon transport, workspace routing order, OCC publication, overlay capture, or isolated pipeline gates change.

Messages And Notifications

Refresh when Message block schema, stream events, system notification semantics, event printing, or message JSONL recorder shape changes.

Support Assets

Refresh search and navigation entries whenever pages, section ids, titles, module labels, or cross-module links change.

Component Gap Sweep #

  • backend/src/providers is now covered by Model Provider Runtime. It was the main missing source module from the original engine/notification/message/provider/prompt scope.
  • backend/src/engine/audit, AgentRunTracker, and PromptReportRecorder are now covered by Observability And Persistence.
  • backend/src/sandbox/provider remains covered separately by Sandbox Tool Dispatch Path to avoid mixing model-provider and sandbox-provider ownership.

Stale Claims #

  • docs/task_center_harness_and_context_engine.html contains historical references to entry_executor; current context-engine recipe files are planner, generator, and evaluator.
  • The same older TaskCenter HTML says executor routing no longer splits by depth; current TerminalToolRouter still removes handoff terminals at nested workflow depth greater than one.
  • The older renderer description says high-priority context can be dropped after medium priority; current XmlPromptRenderer truncates low then medium priority blocks and preserves required/high blocks.
  • A comment in EphemeralAgent.run still mentions a retry path, but run_ephemeral_agent documents that retry-with-fresh-budget no longer exists.
  • A historical sandbox load report mentions public api.shell_batch; current daemon/API source exposes api.v1.shell and harness-local batch helpers.
  • The singular backend/src/provider path does not exist; the model-provider package is backend/src/providers and the sandbox-provider package is backend/src/sandbox/provider.
