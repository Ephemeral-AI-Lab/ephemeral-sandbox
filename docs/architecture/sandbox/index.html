Current code-backed guide to the Rust sandbox substrate: agent-core/crates/eos-sandbox-api, agent-core/crates/eos-sandbox-host, and sandbox/crates/*. It covers LayerStack, overlay, OCC, workspace modes, daemon/background handling, Docker host lifecycle, command sessions, PTY allocation, plugin PPC, and isolated workspaces.

Wiki Contract This directory is the maintained wiki layer over the sandbox source tree. The code checkout is the raw source of truth; these pages synthesize structure, workflows, invariants, and backlinks so a future reader does not have to rediscover the architecture from scratch.

Source Graph §

The current sandbox workspace runtime is best read as a graph of ownership boundaries rather than a stack of folders. The daemon receives tool/API envelopes, routes work into direct LayerStack/OCC paths, per-operation overlay paths, or isolated private scratch, and every durable mutation converges on OCC before LayerStack publishes a new manifest.

importseos-sandbox-api/src/tool_api builds typed daemon payloads for read/write/edit/command/plugin/control/isolation calls and parses daemon responses into public result models. The former grep/glob payload modules were deleted from the Rust facade.
owns-providereos-sandbox-host owns Docker lifecycle, runtime artifact upload, daemon connection, and host-side isolated enter/exit wrappers. Rust sandbox host config is Docker-only today.
registerseos-daemon/src/dispatcher.rs registers built-in workspace ops, command-session ops, isolated lifecycle ops, daemon control ops, and plugin runtime ops.
callsDaemon workspace routing chooses isolated handles first, then direct file fast paths, then overlay-backed execution. Durable mutations still publish through OCC.
owns-stateeos-layerstack owns active manifests, immutable layer directories, leases, merged reads, publish transactions, squash, and flush.
owns-stateeos-isolated and daemon isolated handlers own per-caller private handle maps, namespace holders, veth resources, TTL, persisted handle state, and audit events.
tested-byLive and mock suites cover workspace base, LayerStack/OCC/overlay, background tools, plugins, and isolated workspace behavior under backend/tests/live_e2e_test/sandbox and backend/src/test_runner/tests/mock/sandbox.
